# ドキュメントレビューユーティリティ

Codex CLIを使用してドキュメントをレビューし、結果を返す。

## 実行コマンド

```bash
codex exec --full-auto --sandbox read-only --cd <project_directory> "<request>"
```

## プロンプトのルール

**重要**: codexに渡すリクエストには、以下の指示を必ず含めること：

> 「確認や質問は不要です。具体的な指摘まで自主的に出力してください。」

## ワークフロー

### Step 1: レビュー対象の特定

以下の優先順位でレビュー対象を特定する：

1. `$ARGUMENTS`にファイルパスが指定されている場合 → そのファイルを対象にする
2. 呼び出し元のコンテキストにドキュメントパスや内容が含まれる場合 → それを対象にする
3. 上記いずれも該当しない場合（自然言語で対象未指定）→ **git差分を対象にする**

#### git差分を対象にする場合

```bash
git diff HEAD
```

を実行して差分を取得する。差分が空の場合は `git diff --staged` を試みる。
それも空の場合は `git diff HEAD~1` で直近コミットとの差分を使用する。
取得した差分のうち、`.md` や `.txt` などドキュメントファイルの変更のみを対象とする。

### Step 2: Codex CLIでレビュー実行

対象がファイルパスの場合：

```bash
codex exec --full-auto --sandbox read-only --cd <project_directory> \
  "以下の観点でドキュメントをレビューしてください。

【レビュー観点】
1. 追記部分のみの詳細度が過剰に上がり、ドキュメント全体のバランスを崩していないか
2. 順序性が必然でない場面で見出しにorderが混入していないか
3. 解説意図のメタ補足（例：「項目定義（最小構成）」「バリデーション（最新）」）が入っていないか
4. 改訂履歴を示唆する記述（例：「判定しない方針に変更したため削除」）が残っていないか
5. テキスト校正エラー（例：英単語前後への不要な半角スペース混在）が残っていないか
6. DRY原則に違反していないか
7. YAGNI原則に違反していないか

対象: {$ARGUMENTS の内容 または 呼び出し元から渡された内容}

確認や質問は不要です。具体的な指摘まで自主的に出力してください。"
```

対象がgit差分の場合は、差分内容をプロンプトに直接埋め込む：

```bash
codex exec --full-auto --sandbox read-only --cd <project_directory> \
  "以下の観点でドキュメントの変更差分をレビューしてください。

【レビュー観点】
1. 追記部分のみの詳細度が過剰に上がり、ドキュメント全体のバランスを崩していないか
2. 順序性が必然でない場面で見出しにorderが混入していないか
3. 解説意図のメタ補足（例：「項目定義（最小構成）」「バリデーション（最新）」）が入っていないか
4. 改訂履歴を示唆する記述（例：「判定しない方針に変更したため削除」）が残っていないか
5. テキスト校正エラー（例：英単語前後への不要な半角スペース混在）が残っていないか
6. DRY原則に違反していないか
7. YAGNI原則に違反していないか

【git差分】
{取得したgit diffの内容}

確認や質問は不要です。具体的な指摘まで自主的に出力してください。"
```

### Step 3: 結果を返す

Codexの出力を呼び出し元（ユーザーまたは他のスキル）に返す。
